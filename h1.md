# Läksyt h1

### x) lue ja tiivistä:
- SSH (Secure Shell) mahdollistaa kahden koneen kommunikaation komentoriviltä. SSH-avaimet mahdollistavat salasanattoman yhteyden muodostamisen.
- <b>ssh:n asennus</b>
```
sudo apt-get update
sudo apt-get -y install ssh
```
- <b> ssh:n käynnistäminen järjestelmän käynnistyessä sekä heti käyttöönotto: </b>
```
sudo systemctl enable --now ssh
```
- pub-keyn käyttöönotto näkyy kohdassa b, vaikkakin screenshotti puuttuu ...
- <b>kommenttia:</b> katsottiin tätä vieruskaverin kanssa tunnilla, ssh:n asennus tehtiin hiukan erilaisella komennolla `sudo apt-get install openssh-server` --tämäkin kuitenkin toimi. Onko eroja?

- <b>Ansiblea</b> käytetään automatisointiin ja hallintaan, eli useita koneita voidaan hallita kerralla.
- Ansiblen asennuksessa kävikin ilmi, ettei se ole yhteensopiva SSH:n kanssa jos SSH:lle on annettu passphase: muista jättää tämä kenttä tyhjäksi ssh:ta luodessasi.
- ansiblen asennus toimii komennoilla:
```
sudo apt-get update
sudo apt-get install ansible micro bash-completion tree
```
- lisätyt kommentit micro (tekstieditori) ja bash(tab täyttää lausetta)
- ansiblelle tehdään oma kansio mkdir komennolla: `mkdir ansible`. hostit kirjataan .ini tiedostoon `hosts.ini`
- muokkaamalla ´hosts.ini´ -tiedostoa karsitaan pythonin valitukset pois. lisätään tiedostoon 
```
[all:vars]
ansible_python_interpreter=/usr/bin/python3
```
- käyttöä voidaan nopeuttaa poistamalla extra komentoja: luodaan cfg(configure)-kansio, sisälle 
```
[defaults]
inventory = hosts.ini
```
- site.yml listaa mitkä tietokoneryhmät saavat mitkäkin roolit. roolit tehtävä erikseen omaan kansioonsa ` mkdir -p roles/hello/tasks/`
