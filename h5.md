# Läksyt h5

### x) lue ja tiivistä:

- Gitin ero muihin VCS ohjelmiin on sen "snapshot" mallinen toimintatapa. Monet mallit säilyttävät tietonsa muutoksista listana, kun taas git ottaa kuvankaappauksia jokaisesta projektin tilasta.
- Gitin operaatiot ovat suurimmilta osiltaan lokaaleja, eli ne pyörivät koneellasi. Muutoksien tekemiseen tarvitaan harvoin internet yhteyttä tai toista konetta, kun projektihistoria haetaan suoraan oman koneesi tietokannasta. Paikallisiin committeihin ei tarvitse internetiä, ainoastaan projektin lataamiseen verkkoon tai sieltä vedettäessä.
- Git ylläpitää eheyttä SHA-1 tiivisteen muodossa, joka muuttuu joka kerta kun tiedostoon tehdään muutoksia.

`git add --all && git commit; git pull && git push` - komento selittyy näin:

- `git add --all` - git lisätään kaikki (projektin kaikki tiedostot)
-  `&&` on putkitus, jatkaa komentoa yhdeltä riviltä eli ei tarvitse kirjoittaa monta eri komentoa
-  `git commit; git pull` vedetään committiin esim. netistä viimeisin versio.
-  `&& git push` - putkitus, ja työnnetään projekti kohteeseen.
