### x) lue ja tiivistä:

<b>sudo without password</b>

- ansible tarvitsee root -käyttöoikeudet myös orjakoneelle
- `sudoless` -komento on hyvödyllinen, sillä se poistaa salasanan tarpeen totetuttaessa korkeampien oikeuksien käskyjä
- asennus toimii tekemällä ensin ryhmä "sudoless", ja lisäämällä sinne käyttäjä
```
$ sudo adduser antero
$ sudo groupadd sudoless
$ sudo adduser antero sudoless
```
- luodaan käyttäjä antero, luodaan ryhmä sudoless, lisätään antero ryhmään
- <b> jos sudo hajoaa:</b>
- `sudo -i` -komennolla päästään root käyttäjäksi, jolla on oikeudet tehdä mitä tahansa koneella
<b> sudoers-sääntö</b>
- sudoless ryhmän tulee päästä käyttämään sudo-komentoa ilman salasanaa
- ´visudo´ takaa sudoers-tiedoston turvallisen muokkaamisen validoimalla tiedoston syntaksin ennen tallentamista. komento:
  ```
  $ sudo visudo /etc/sudoers.d/sudoless

  ```
- .d tarkoittaa directory(kansio), jossa on config tiedostoja. directoryyn lisätään komento
  ```
  %sudoless ALL = (ALL) NOPASSWD: ALL
  ```
- % tarkoittaa ryhmää (ryhmä sudoless), ALL kaikki koneet, (ALL) olla mikä vain käyttäjä, NO PASSWD ei salasanaa, viimeinen ALL kaikki komennot sallittu
- konfiguroinnin jälkeen kirjaudu ulos käyttäjältäsi ja takaisin sisään jotta muutokset astuvat voimaan
 
  <b> passwordless sudo with ansible</b>

- varmista että salasanaton sudo toimii manuaalisesti ennen ansiblen konfigurointia
- lisätään aikaisempaan ansible -konfiguraatioon rooli nimeltä antero, ja lisätään sinne kansio tasks, ja sinne tiedosto main.yml. sen sisälle:
```
- group:
    name: "sudoless"
    state: present
- user:
    name: "antero"
    state: present
    groups: ["sudoless", "sudo", "adm"]
- authorized_key:
    user: "antero"
    key: "ssh-ed25519 U2VlIHlvdSBhdCBUZXJvS2FydmluZW4uY29tIQ== tero@example.com"
- copy:
    dest: "/etc/sudoers.d/sudoless"
    content: "%sudoless ALL = (ALL) NOPASSWD: ALL\n"
    owner: "root"
    group: "root"
    mode: "0644"
```
- `site.yml` tiedostoon lisätään `become: true`--aktivoi oikeuksien eskalaation.
